## Architecture

SwiftUI app (iOS 16+) with three observable managers injected as `@EnvironmentObject` from the root `BirthdayReminderApp`.

**Managers** (business logic, no UI):
- `FriendsManager` — CRUD for `Friend` objects, persisted to `UserDefaults` as JSON
- `SettingsManager` — two user preferences (`defaultReminderDays`, `upcomingDaysToShow`), persisted to `UserDefaults` key-value
- `NotificationManager` — static enum; schedules up to 60 non-repeating notifications two years in advance to stay under iOS's 64-notification cap; must be called after any mutation to friends or settings
- `ContactsManager` — static enum; async `CNContactStore` wrapper that fetches contacts on a background GCD queue and returns `Friend` objects on the main thread

**Navigation**: Custom bottom tab bar in `MainTabView` keeps all three tab views in memory simultaneously (opacity/hit-testing toggling). `FriendsView` uses `NavigationStack` for push navigation to `FriendDetailView`. Modals use `.sheet()`.

**Data flow**: User action → manager method → `@Published` update → `UserDefaults` persist → `NotificationManager.scheduleAll()` called.

## Key Conventions

- `Friend` uses a custom `Codable` init to handle migration from `reminderDays: Int?` (old) to `reminderDays: [Int]` (current). Don't break this.
- `AvatarColor` is assigned deterministically via name hash — don't make it random.
- `FriendFormView` uses an enum mode (`.add` / `.edit(Friend)`) — this pattern should be followed for any future form views.
- Notification identifiers are prefixed `"birthday_"` so they can be selectively cleared without touching other app notifications.
- `AppTheme` in `Theme/AppTheme.swift` is the single source of truth for colors. Accent/sage green: `#6B8F71`.
- All icon sizes are generated by `generate_icons.py` from a 1024×1024 master — edit the script and re-run rather than editing individual PNGs.
